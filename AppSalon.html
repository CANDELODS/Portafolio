Arquitectura

MVC

Patrón de arquitectura de software que nos permite la separación de obligaciones de cada parte del código, separa la lógica de la vista del proyecto.

POO

La Programación Orientada a Objetos (POO) es un paradigma de desarrollo que organiza el código en objetos. Estos objetos son instancias de clases, que agrupan atributos (datos) y métodos (funciones) para representar entidades del mundo real.

Active Record

Patrón de arquitectura (para los datos) usado para apps que almacenan datos en bases de datos, sobre todo cuando se usa CRUD, es el responsable de trabajar con los datos, en un MVC sería la M (Modelo).

Router

Es el encargado de registrar todas las URLS o Endpoints que soporta nuestro sitio web, cuando un usuario accede a una URL el router ya sabe que controlador y modelo llamar.

API

(Interfaz de Programación de Aplicaciones, por sus siglas en inglés) es un conjunto de reglas y herramientas que permite a diferentes aplicaciones o sistemas comunicarse entre sí, en este proyecto se creó una para poder manejar los servicios (cortes, peinados, etc) y hacer su CRUD más facil y escalable.
